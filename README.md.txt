# 🤖 AI Workflow Audit & Setup

## 📊 Workflow Audit

| Task | Classification | Rationale |
| :--- | :--- | :--- |
| Attend lectures | Just Me | Requires real-time personal attention and active participation. |
| Solve university assignments | Just Me | Core learning and academic integrity require personal execution. |
| Take lecture notes | Collaborate with AI | AI structures and cleans up notes while I verify key takeaways. |
| Revise programming concepts | Collaborate with AI | AI explains complex topics while I practice implementing them. |
| Build UI layouts | Collaborate with AI | AI generates initial widget layouts, which I customize and refine. |
| Research APIs & packages | Collaborate with AI | AI accelerates package discovery, but implementation requires verification. |
| Debug Flutter errors | Delegate to AI with Review | AI identifies syntax and logic bugs, which I review before applying. |
| Prepare presentation slides | Delegate to AI with Review | AI outlines slide structures, and I refine the technical content. |
| Write README files & docs | Delegate to AI with Review | AI drafts initial documentation, which I edit for accuracy. |
| Generate unit test cases | Delegate to AI with Review | AI writes test templates, and I validate edge cases. |
| Write Git commit messages | Fully Automate | AI consistently formats conventional commit messages. |
| Plan weekly schedule | Fully Automate | AI organizes deadlines and tasks into an optimized schedule. |

## 🎯 Target Tasks & Success Definitions

### Task 1: Debugging Flutter Errors
* **Success Definition:**
  * Root cause of the error correctly identified.
  * Clear explanation of why the fix works.
  * Bug is resolved without introducing regression errors.

### Task 2: Building Responsive Flutter UI
* **Success Definition:**
  * UI adapts seamlessly across different screen dimensions.
  * Code follows clean, modular, and reusable widget practices.
  * Matches design and functional requirements.

### Task 3: Generating Project Documentation
* **Success Definition:**
  * Clear, concise, and grammatically accurate language.
  * Well-structured formatting using standard Markdown headers and lists.
  * Accurately reflects project architecture and setup steps.
